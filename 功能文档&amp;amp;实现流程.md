# AI语音绘图工具 功能开发文档
**版本号**：v1.0  
**更新日期**：2026-06-12  
**文档状态**：正式发布  
**开发参考**：按阶段顺序开发，每阶段完成并通过验收后再进入下一阶段

**项目说明**：AI 语音绘图工具
请开发一款纯语音控制的绘图工具。

要求：用户不能使用鼠标或键盘，仅通过语音指令完成绘图创作。请综合考虑指令理解的准确性与容错性、语音到绘图操作的响应延迟，以及复杂指令的拆解与执行能力。实现应用的同时，请额外提交一份设计文档，记录你计划支持哪些指令能力，最终实现了哪些，以及未完成部分的原因说明。

## 一、文档概述
### 1.1 文档目的
本文档详细定义了AI语音绘图工具的所有功能规格、语音指令集、验收标准和技术实现要点，作为开发团队的唯一开发依据，确保功能实现的一致性和完整性。

### 1.2 适用范围
- 前端开发工程师：负责绘图渲染、语音采集、UI交互
- 后端开发工程师：负责指令调度、状态管理、AI服务对接
- AI工程师：负责语音识别、指令解析、语音合成优化
- 测试工程师：负责功能测试和验收

### 1.3 开发原则
- **先核心后扩展**：优先实现第一阶段核心功能，确保基础体验
- **语音优先**：所有功能必须支持纯语音控制，UI仅作为辅助显示
- **低延迟**：所有指令从语音输入到绘图完成的总延迟≤500ms
- **高容错**：支持口语化表达、轻微口误、模糊指令补全

## 二、产品整体架构
```
用户语音输入 → 语音采集 → 流式ASR识别 → 指令解析引擎 → 指令队列调度 → 前端绘图渲染 → 语音合成反馈
                          ↑                         ↓
                      上下文管理 ←-------------- 状态同步
```

## 四、详细功能规格说明
### 4.1 系统交互模块
#### F001 唤醒与休眠
- **功能描述**：通过语音唤醒和休眠工具，避免误触发
- **支持指令**：
  - 唤醒："小画"、"绘图助手"
  - 休眠："退出"、"关闭"、"休息吧"
- **技术实现**：
  - 使用Web Audio API持续监听麦克风输入
  - 本地轻量唤醒词模型（如Porcupine）实现低功耗唤醒
  - 唤醒后显示语音输入指示器，休眠后隐藏
- **异常处理**：连续3次未识别到有效指令自动休眠

#### F002 语音状态反馈
- **功能描述**：所有操作执行后通过语音告知用户结果
- **反馈规则**：
  - 成功操作：简短确认音+语音提示（如"已新建画布"）
  - 错误操作：清晰错误说明+引导（如"未识别指令，请再说一遍"）
  - 长时操作：进度播报（如"正在执行第一步，绘制圆形"）
- **技术实现**：对接TTS服务，生成语音后自动播放
- **验收标准**：所有操作都有对应的语音反馈，反馈延迟≤200ms

### 4.2 画布操作模块
#### F003 新建画布
- **功能描述**：创建一个新的空白画布
- **支持指令**：
  - "新建画布"、"创建新画板"、"开一张新图"
  - "新建A4画布"、"新建正方形画布"
- **默认参数**：宽度800px，高度600px，白色背景
- **技术实现**：调用Fabric.js的Canvas构造函数
- **异常处理**：如果已有未保存内容，语音提示"是否保存当前画布？"

#### F004 清空画布
- **功能描述**：清除画布上的所有内容
- **支持指令**：
  - "清空画布"、"擦掉全部内容"、"重置画板"
- **技术实现**：调用Fabric.js的clear()方法
- **异常处理**：语音确认"确定要清空画布吗？"，用户回答"确定"后执行

### 4.3 基础绘图模块
#### F007 绘制圆形
- **功能描述**：在指定位置绘制指定大小的圆形
- **支持指令**：
  - "画一个圆"、"画一个正圆"、"画一个椭圆"
  - "在中间画一个大圆"、"在右上角画一个小圆"
  - "画一个半径50的圆"
- **参数提取**：
  - 位置：左上角、右上角、左下角、右下角、中间
  - 大小：大、中、小（对应半径100、50、25）
  - 半径：具体数值（像素）
- **技术实现**：创建Fabric.Circle对象，添加到画布
- **默认参数**：位置居中，大小中等，当前画笔颜色，无填充

### 4.4 样式设置模块
#### F010 画笔颜色
- **功能描述**：设置后续绘制图形的画笔颜色
- **支持指令**：
  - "画笔换成红色"、"改成蓝色"、"用绿色画"
  - "填充黄色"、"轮廓黑色"
- **支持颜色**：红、橙、黄、绿、蓝、紫、黑、白
- **技术实现**：修改全局画笔样式变量，后续绘制自动应用
- **验收标准**：设置颜色后绘制的图形使用新颜色

## 五、语音指令集完整定义
### 5.1 指令语法规则
所有指令遵循以下通用语法：
```
[修饰词] + 动作动词 + [对象] + [参数]
```
- 修饰词：如"在中间"、"大的"、"红色的"
- 动作动词：如"画"、"改"、"移"、"删"、"保存"
- 对象：如"圆"、"矩形"、"画布"、"文字"
- 参数：如"半径50"、"向上移100"、"透明度50%"

### 5.2 同义指令映射表
| 标准指令 | 支持的口语化表达 |
|----------|------------------|
| 删除图形 | 擦掉、去掉、删除、清除 |
| 移动图形 | 挪一下、移过去、放到 |
| 放大图形 | 变大一点、调大、放大 |
| 撤销操作 | 退回去、上一步、撤销 |
| 保存图片 | 存一下、导出、保存 |

### 5.3 复杂指令拆解示例
**用户指令**："在画布下方画一个蓝色大圆，再复制两个，全部缩小一半，移到中间并排"
**拆解步骤**：
1. 设置画笔颜色为蓝色
2. 在画布下方绘制一个大圆
3. 复制该圆形两次
4. 选中所有三个圆形
5. 将它们缩小到原来的50%
6. 将它们水平居中对齐
7. 设置它们之间的间距相等

## 六、技术实现要点
### 6.1 前端绘图实现
- 所有绘图操作通过Fabric.js API执行，禁止直接操作Canvas上下文
- 维护全局画布对象`canvas`和当前选中对象`selectedObject`
- 实现指令执行队列，确保指令按顺序执行
- 本地缓存最近10步操作历史，支持撤销和恢复

### 6.2 语音识别实现
- 使用流式ASR接口，边说话边返回文本
- 自定义热词表，添加所有绘图相关术语
- 实现静音检测，用户停止说话后自动结束识别
- 对识别结果进行后处理，纠正常见识别错误

### 6.3 指令解析实现
- 第一阶段：使用正则表达式匹配预定义指令模板
- 第二阶段：使用大模型将自然语言转换为结构化JSON指令
- 维护会话上下文，结合历史指令理解当前意图
- 模糊指令时生成确认问题，通过语音询问用户

### 6.4 状态管理实现
- 前端维护完整的画布状态和用户操作历史
- 后端维护会话状态，包括当前用户、画布ID、历史指令
- 使用Redis缓存会话状态，过期时间30分钟
- 所有状态变更通过WebSocket实时同步

## 七、验收标准
### 7.1 功能验收标准
- 所有功能必须支持纯语音控制，无需鼠标或键盘
- 语音指令识别准确率≥95%（标准普通话）
- 简单指令从语音输入到绘图完成的总延迟≤500ms
- 复杂指令拆解准确率≥90%
- 所有操作都有对应的语音反馈

### 7.2 性能验收标准
- 画布支持最多100个图形对象，操作流畅无卡顿
- 语音采集和传输延迟≤100ms
- ASR识别延迟≤200ms
- 指令解析延迟≤100ms
- 绘图渲染延迟≤100ms
